# Planning

This folder is the source of truth for project scope, current status, and delivery workflow.

## Current Snapshot

- Product status: local MVP is implemented and builds cleanly
- UI direction: minimal, old-school, Hacker News-style, information-first
- Team workflow: basic GitHub Flow is in place
- CI status: lint, typecheck, and production build run in GitHub Actions
- Main gaps: automated tests, stronger PR discipline, preview/staging workflow, deployment checklist

## Documents

- `PROJECT.md`: product scope, constraints, and design direction
- `REQUIREMENTS.md`: implemented requirements and next quality/process requirements
- `ROADMAP.md`: milestone-level delivery plan
- `STATUS.md`: what exists now, what is missing, and what is next
- `WORKFLOW.md`: two-person development workflow, PR process, and CI/testing plan
